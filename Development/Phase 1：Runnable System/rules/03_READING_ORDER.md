# 03 — 读取顺序与工作流程

---

## 启动流程（每次交互开始时）

AI 必须按以下顺序读取：

```
Phase 0: 定位项目根目录
  → 确认当前工作目录为本项目目录

Phase 1: 建立行为准则（必须全读）
  → rules/00_CORE_PRINCIPLES.md
  → rules/01_ROLE_DEFINITION.md
  → rules/02_OUTPUT_FORMAT.md
  → rules/03_READING_ORDER.md  （本文件）
  → rules/04_CONSTRAINTS.md

Phase 2: 了解数据全貌
  → data/ 目录（若存在且有内容）

Phase 3: 了解历史上下文
  → sessions/ 中的最近会话记录（若存在）

Phase 4: 了解当前状态
  → summary.md（若存在）
```

---

## 按需读取（根据用户意图触发）

| 用户意图 | 应读取的数据 |
|----------|-------------|
| 涉及身体健康 | `data/health/` |
| 涉及学习/技能 | `data/learning/` |
| 涉及时间安排 | `data/schedule/` |
| 涉及目标计划 | `data/goals/` |
| 涉及情绪/心理 | `data/mind/` |
| 用户说"看看我最近" | Phase 2 + 3 + 4 全量 |
| 用户说"给点建议" | 先跑 Phase 2 + 3 + 4，再做偏差分析 |

---

## 产出流程（分析完成后）

```
Step 1: 对照核心守则和约束条件自检
  → 每条结论都有数据依据吗？
  → 有标注不确定性吗？

Step 2: 在对话中回应用户（分析、建议、聆听、追问）

Step 3: 更新 summary.md

Step 4: 归档到 sessions/（必定执行）
  → 无论对话长短，必须归档
```

---

## 关键原则

- **先读后说**：在任何分析输出前，确保已读取 Phase 1 全部 + 必要数据。
- **缺数据不强行分析**：如果对应数据目录为空或不存在，明确告知用户需要先补充哪些数据。
- **每次交互归档**：无论对话长短，结束后必须在 `sessions/` 中留下记录。
