# 02 — 输出格式规范

---

## 四种输出类型

AI 在此项目中的输出分为四类，每类有固定结构：

---

### 类型一：建议（Suggestion）

适用场景：基于数据给出行动建议时。

```markdown
## [建议] 建议标题

**数据依据**：[引用的数据文件路径 + 具体条目]

**当前状态**：[一句话描述现状]

**建议行动**：
1. [具体步骤1]
2. [具体步骤2]

**预期效果**：[如果执行，预期会有什么变化]

**检验标准**：[如何判断建议是否有效]

**时间窗口**：[建议在什么时间范围内执行]
```

---

### 类型二：警告（Warning）

适用场景：发现时间冲突、健康风险、进度严重滞后等需要警示的情况。

```markdown
## ⚠️ [警告] 警告标题

**严重程度**：🔴 高 / 🟡 中 / 🟢 低

**数据依据**：[引用的数据文件 + 具体条目]

**冲突/风险描述**：[说明具体的问题]

**紧迫性**：[为什么需要现在关注]

**建议应对**：[具体的应对方案]

**不处理的后果**：[如果不理会，可能发生什么]
```

---

### 类型三：纠错（Correction）

适用场景：发现用户的行为/认知与数据记录存在矛盾时。

```markdown
## [纠错] 纠错标题

**矛盾描述**：[说明矛盾点]

**数据A**：[来源 + 内容]
**数据B**：[来源 + 内容]

**冲突分析**：[为什么这两条数据互相矛盾]

**修正建议**：[建议更新哪条数据 / 调整哪个认知]
```

---

### 类型四：提醒（Reminder）

适用场景：即将到来的截止日期、承诺事项、周期性检查等。

```markdown
## [提醒] 提醒标题

**提醒事项**：[需要关注的事]

**关联数据**：[数据来源]

**时间节点**：[具体日期 / 时间]

**距现在**：[还有多少天/小时]

**建议动作**：[该做什么]
```

---

## 通用规则

1. 每条建议/警告/纠错/提醒都必须标注**数据依据**。
2. 若涉及时间，统一使用 `YYYY-MM-DD HH:MM` 格式。
3. 输出到 `Guidance/` 的文件时，文件名需包含类型前缀和日期：如 `suggestion_2026-07-01_增加运动频率.md`。
4. 对话中的输出可以简化，但核心字段（数据依据、具体行动）不可省略。
